# 評価指標の説明

## 1. 概要

本研究における評価指標は、過去期間（2015-2018年）に基づいて生成したランキングが、将来期間（2019-2024年）の実際の技術動向と対応するかを定量的に測定するために設計されている。

---

## 2. 主要評価指標

### 2.1 U-Netの順位（Rank of U-Net）

**定義：** 各ランキング手法において、U-Net（またはU-Net系手法）が何位にランクされるか。

$$\text{Rank}_{m}^{(\text{method})} = \text{rank of } m \text{ in the ranked list produced by } \text{method}$$

**解釈：**
- 順位が低い（上位）ほど、その手法が U-Net を早い段階で候補として識別できたことを示す。
- 提案手法（GapScore）がベースラインより低い順位（上位）を出力した場合、候補抽出の精度が高いと判断する。

---

### 2.2 Hit@k

**定義：** 上位 $k$ 件のランキング中に U-Net 系手法が1件以上含まれるか（二値指標）。

$$\text{Hit@}k = \mathbb{1}\left[\exists m \in \text{Top-}k : m \in \text{UNetVariants}\right]$$

ここで $\text{UNetVariants} = \{\text{U-Net}, \text{UNet}, \text{nnU-Net}, \text{U-Net++}\}$。

**計算する k の値：** 1, 3, 5, 10

**解釈：**
- Hit@1 = 1：U-Net が1位にランクされた（最も良い結果）
- Hit@3 = 1, Hit@1 = 0：U-Net が上位3位以内にランクされた
- Hit@10 = 0：U-Net がどの手法でも上位10件に入らなかった（検出失敗）

---

### 2.3 B0将来期間の出現率増加（Future Growth）

**定義：** B0（製造欠陥検出）において、U-Net の出現率が過去期間から将来期間にかけてどの程度増加したか。

**絶対値増加：**
$$\Delta r_{B,m} = \text{rate}_{B,m}^{\text{future}} - \text{rate}_{B,m}^{\text{past}}$$

**比率増加：**
$$\rho_{B,m} = \frac{\text{rate}_{B,m}^{\text{future}}}{\text{rate}_{B,m}^{\text{past}} + \varepsilon}$$

**解釈：**
- $\Delta r > 0$：将来期間で出現率が増加（正の成長）
- $\rho > 1$：将来期間の出現率が過去期間より高い
- この指標が正である場合、後ろ向き評価の「正解」として使用する。

---

### 2.4 ドメイン間コサイン類似度（Domain Similarity）

**定義：** A0とB0の代表ベクトル間のコサイン類似度。

$$\text{sim}(A_0, B_0) = \cos\!\left(\bar{\mathbf{e}}_{A_0}, \bar{\mathbf{e}}_{B_0}\right) = \frac{\bar{\mathbf{e}}_{A_0} \cdot \bar{\mathbf{e}}_{B_0}}{\|\bar{\mathbf{e}}_{A_0}\| \|\bar{\mathbf{e}}_{B_0}\|}$$

**計算するバリエーション：**

| 比較対象 | 埋め込みタイプ | 意味 |
|---|---|---|
| A0全体 vs B0全体 | full / problem / solution | ドメイン全体の類似性 |
| A0過去 vs B0過去 | full / problem / solution | 同時期のドメイン類似性 |
| A0将来 vs B0将来 | full / problem / solution | 将来期間のドメイン類似性 |
| A0_U-Net vs B0全体 | problem | U-Net課題群とB0の課題類似性 |
| A0_U-Net vs B0_U-Net | problem | U-Net採用特許群同士の類似性 |
| A0過去_U-Net vs B0過去 | problem | 過去期間の課題類似性（GapScoreの分子） |

**解釈：**
- 課題文脈埋め込み（problem）での類似度が高いほど、A0で有効な技術がB0でも有効である可能性が高い。
- 解決手段文脈（solution）より課題文脈（problem）の類似度を重視するのが、本研究の特徴的な仮説である。

---

### 2.5 Method Gap（手法別出現率差）

**定義：** 手法 $m$ の A0 と B0 における出現率の差。

$$\text{gap}_m = \text{rate}_{A,m} - \text{rate}_{B,m}$$

**比率版：**
$$\text{ratio}_m = \frac{\text{rate}_{A,m}}{\text{rate}_{B,m} + \varepsilon}$$

**解釈：**
- gap > 0：手法 $m$ はA0でより多く使われている（B0への展開余地あり）
- gap < 0：手法 $m$ はB0でより多く使われている（すでに普及済み）
- gap = 0 かつ rate_A > 0：両ドメインで同程度使われている

---

## 3. 評価の解釈フレームワーク

```
過去期間のランキング結果
    ↓
U-Netは上位にランクされたか？（Rank / Hit@k）
    ↓
将来期間でB0のU-Net出現率は増加したか？（Future Growth）
    ↓
[両者が対応する] → 後ろ向き評価において候補抽出の可能性を示す結果
[両者が対応しない] → 手法の限界として考察する
```

|---|---|---|---|---|---|
| Baseline1: 頻度順 | ? | ? | ? | ? | ? |
| Baseline2: Gapのみ | ? | ? | ? | ? | ? |

提案手法がベースラインよりも低い順位（上位）を示す場合、課題文脈の分離と GapScore による重み付けが候補抽出精度の向上に寄与することを示す根拠となる。

---

## 5. 注意事項

- 評価対象手法数が少ない（15手法程度）ため、統計的検定の適用には限界がある。
- Hit@k は1件のデータポイント（U-Net）に対する評価であるため、偶然性の影響を考慮して解釈する必要がある。
- B0の将来期間出現率増加は実際の特許データに基づくが、特許出願のタイムラグや分類の粒度によって変動しうる。
- 本指標体系は研究手法の有効性を示す参考指標であり、専門家による定性的評価と組み合わせて解釈することが望ましい。
