# 1. 研究背景・問題意識

異分野技術探索における課題

---

# 研究背景

## なぜ異分野技術探索が必要か

- 企業・研究機関は **自分たちの分野の外** にある技術手法を参照することで，新しい問題解決の糸口を得る場合がある
- 例：画像解析，材料処理，洗浄，検査，制御……**対象物が違っても課題が近い**ことがある
- 特許文書は技術の名称・課題・解決手段・効果を含む **豊富な知識源**

## 特許文書を使う利点

| 情報の種類 | 内容 |
|---|---|
| タイトル | 発明の概要 |
| 要約（Abstract） | 目的・構成の要約 |
| クレーム（Claims） | 権利範囲・技術構成の詳細 |

→ キーワード検索だけでなく，**課題と解決手段の関係**を読み取れる

---

# 問題意識

## 全文類似の限界

<div class="highlight">
全文をまとめてベクトル化すると，課題・対象物・解決手段・効果が<strong>同一の表現空間に混在</strong>し，候補が抽出された理由を説明しにくい
</div>

### 具体的な問題

1. **語彙の違い問題**：「医療画像」「製造欠陥」「半導体基板」など分野固有語が類似度に強く影響
2. **理由の不明瞭さ**：全文として似ていても，「課題が近いから？」「対象物の用語が似ているから？」かが不明
3. **有望候補の見落とし**：対象物や分野名は違うが課題が近い候補が，全文類似だけでは上位に現れにくい

<div class="warn-box">
💡 専門家が候補を確認する際，「<strong>なぜこの候補を見るべきか</strong>」を理解できる形で提示することが重要
</div>

# 2. 研究目的と提案手法の概要

---

# 研究目的

## 本研究が解く問い

> 「ある**ドメインA**で広く用いられている技術手法は，類似した課題構造を持つ**ドメインB**において，将来的に出現増加する**候補**となりうるか？」

## 本研究のアプローチ

特許文書を `problem_context`（**課題文脈**）と `solution_context`（**解決手段文脈**）に**分離**し，

- 課題が近い異分野間において
- 対象分野では **低出現または未使用** である技術手法・解決手段を
- **専門家が確認すべき候補** として抽出する

<div class="warn-box">
⚠ 本研究は<strong>技術導入可能性を証明するものではない</strong>。専門家確認候補を絞り込む<strong>スクリーニング手法</strong>として位置づける
</div>

---

# 提案手法の全体フロー

```
特許文書
(Title / Abstract / Claims)
         │
         ▼
  ┌─────────────────────────────────────────────────────┐
  │  LLMによる文脈抽出                                   │
  │  problem_context   ← 「何の課題を解いているか」       │
  │  solution_context  ← 「どのように解いているか」       │
  │  technical_means   ← 具体的手法名・構成要素          │
  │  target_object     ← 対象物                         │
  │  effect_context    ← 効果・改善点                   │
  └─────────────────────────────────────────────────────┘
         │
         ▼
  PatentSBERTa によるベクトル化
  （文脈ごとに768次元ベクトルを生成）
         │
         ▼
  課題文脈類似度 × Method Gap × 頻度重み × 低出現ボーナス
  ──────────────────────────────────────────────────────
                   GapScore
         │
         ▼
  専門家確認候補ランキング → LLM支援付き著者確認評価
```

# 3. 提案手法の詳細

GapScoreの計算方法

---

# 文脈抽出：課題と解決手段の分離

## 抽出項目の定義

| フィールド | 意味 | 例（医療画像解析） |
|---|---|---|
| `problem_context` | 何の課題を解いているか | 「腫瘍領域の精密な分割が困難」 |
| `solution_context` | どのように解いているか | 「U-Netによるエンコーダ-デコーダ構造を用いた」 |
| `technical_means` | 具体的な技術手段・手法名 | U-Net, GAN, Transformer |
| `target_object` | 技術が適用される対象物 | CT画像，MRI，病変組織 |
| `application_field` | 応用分野 | 医療診断支援，放射線科 |
| `effect_context` | 得られる効果 | セグメンテーション精度の向上 |

### 分離の意義

<div class="highlight">
全文類似では「医療画像 ↔ 欠陥画像」の<strong>対象物の違い</strong>が影響するが，<code>problem_context</code>に絞れば「<strong>画像中の異常領域を検出する</strong>」という課題の近さを捉えやすい
</div>

---

# ベクトル化と類似度計算

## PatentSBERTa によるベクトル化

特許特化型言語モデル **PatentSBERTa**（Sentence-BERTの特許ドメインFine-tune版）を使用

$$\mathbf{e}_{i}^{\text{problem}} = \text{PatentSBERTa}(\text{problem\_context}_i) \in \mathbb{R}^{768}$$

**ドメイン代表ベクトル**（各ドメインの平均ベクトル）：

$$\bar{\mathbf{e}}_{A_0}^{\text{problem}} = \frac{1}{|A_0|}\sum_{i \in A_0} \mathbf{e}_{i}^{\text{problem}}$$

**コサイン類似度**でドメイン間の課題構造の近さを定量化：

$$\text{sim}(A_0, B_0) = \cos\!\left(\bar{\mathbf{e}}_{A_0}^{\text{problem}},\ \bar{\mathbf{e}}_{B_0}^{\text{problem}}\right)$$

> フォールバック：`sentence-transformers/all-MiniLM-L6-v2`

---

# GapScore の定義

## Method Gap（手法ごとの出現率差）

手法 $m$ について，A0とB0の過去期間の出現率差：

$$\text{gap}_m = \text{rate}_{A,m} - \text{rate}_{B,m}$$

$$\text{rate}_{A,m} = \frac{|\{d \in A_0^{\text{past}} : m \in d\}|}{|A_0^{\text{past}}|}$$

## GapScore（提案手法）

$$\text{GapScore}_m = \underbrace{\text{sim}^{\text{prob}}(A_0^{\text{unet}}, B_0)}_{\text{課題類似度}} \times \underbrace{\frac{\text{gap}_m}{\max_j \text{gap}_j}}_{\text{正規化Gap}} \times \underbrace{\log(1 + N_{A,m})}_{\text{頻度重み}} \times \underbrace{\min\!\left(\frac{1}{\text{rate}_{B,m} + \varepsilon},\ C\right)}_{\text{低出現ボーナス}}$$

| 項 | 意味 |
|---|---|
| 課題類似度 | A0のU-Net特許群の課題ベクトルとB0全体のコサイン類似度 |
| 正規化Gap | A0−B0の出現率差（B0でまだ少ない手法を高評価） |
| 頻度重み | A0での出現件数の対数（稀すぎる手法にペナルティ） |
| 低出現ボーナス | B0出現が少ない手法ほど高スコア（探索価値が高い） |

---

# 候補タイプの定義

<div class="two-col">

<div>

## 近接解決策型（実験1・主分析）

- 課題文脈・解決手段文脈の**両方が比較的近い**異分野間の候補
- 例：医療画像解析 → 製造欠陥検出
  - 共通課題：「画像中の異常・欠陥領域を検出・分割する」
  - 対象物は違うが手法が転用できる

</div>

<div>

## 異質解決策型（実験2・補助分析）

- 課題文脈に**一定の関連**はあるが，対象物・材料・工程・用途分野が**大きく異なる**候補
- 例：洗浄・除去系技術 → 半導体洗浄
  - 「洗浄・残渣除去」という課題は関連
  - 対象物や工程は大きく異なる

</div>
</div>

<br>

<div class="highlight">
課題文脈類似度（Problem Similarity）が高く，解決手段文脈類似度（Solution Similarity）の差が大きい組み合わせが<strong>候補抽出の対象</strong>
</div>

# 実験1の設計

## 研究目的

医療画像解析で先行して現れたU-Netが，製造欠陥検出分野における**専門家確認候補として抽出できるか**を確認する（後ろ向き評価）

## ドメイン定義

| 記号 | 分野 | 役割 |
|---|---|---|
| **A0** | 医療画像解析（Medical Image Analysis） | 参照元ドメイン（技術供給源） |
| A1 | A0のU-Net関連集合 | U-Net出現確認用 |
| **B0** | 製造欠陥検出（Manufacturing Defect Detection） | 対象ドメイン（候補探索先） |
| B1 | B0のU-Net関連集合 | U-Net出現確認用 |

## 期間分割

| 期間 | 年範囲 | 用途 |
|---|---|---|
| **過去期間（Past）** | 2015〜2018年 | 候補抽出・ランキング生成に使用 |
| **将来期間（Future）** | 2019〜2024年 | 後ろ向き評価（出現増加の確認）に使用 |

---

# 実験1のデータセット

## データ概要（Orbis IPより取得）

| データセット | 公開公報数 | ファミリー数 |

| A0（医療画像解析・全体） | 54,100件 | 30,093件 |
| B0（製造欠陥検出・全体） | 44,364件 | 28,450件 |

## U-Net抽出条件

- Title または Abstract に **"U-Net", "UNet", "nnU-Net", "U-Net++", "UNet++"** を含むもの
- PublicationNumber（公開公報番号）が存在するレコードを1件としてカウント
- Family Identifier で重複除去

<div class="result-box">
⚙ 技術的特記事項：xlsxのCentral Directoryが欠損していたため，Local File Headerを直接解析してsharedStrings.xml（展開後 ≈900MB）をストリーム解析（32MBチャンク）
</div>

# 実験1：候補抽出の流れ

## データ処理パイプライン

```
A0_data.csv / B0_data.csv（Orbis IPから抽出）
         ↓
01. 前処理：Publication / Family レベルのCSV作成
         ↓
02. 手法抽出：U-Net, YOLO, GAN, Transformer 等の出現フラグ付与
         ↓
03. 文脈抽出：problem_context / solution_context の分離（ルールベース + LLM補完）
         ↓
04. ベクトル化：PatentSBERTa による768次元ベクトル化
         ↓
05. 類似度計算：A0 vs B0 のコサイン類似度（課題・解決手段・全文）
         ↓
06. Method Gap 計算：期間別・手法別の出現率差
         ↓
07. GapScore ランキング：過去期間のみの情報でランキング生成
         ↓
08. 後ろ向き評価：U-Netの順位確認 + B0将来期間での出現増加確認
         ↓
09. ベースライン比較：提案手法 vs 5種のベースライン
```

<div>

**全文類似上位ベースライン（30件）**

# 実験1：まとめ

## 得られた知見

1. **U-Netの出現傾向**：医療画像解析（A0）で先行増加，製造欠陥検出（B0）では将来期間に出現確認 → 後ろ向き評価の前提が成立

2. **候補抽出結果**：過去期間の情報のみで，U-Netを全体**7位**（上位10件以内）・具体的手法**2位**にランキング

3. **ベースライン比較**：
   - 全文類似上位ベースラインは×が多く，専門家確認候補として**弱い候補が混入しやすい**
   - 提案手法は課題文脈ベースで×が少なく，**候補抽出理由を説明しやすい**

4. **後ろ向き評価の意義**：B0の将来期間では0件→314件へ増加しており，過去期間での抽出が実際の出現増加と対応

<div class="result-box">
✅ 近接解決策型の主分析として，課題文脈・解決手段文脈分離に基づく候補抽出が，専門家確認候補を整理するスクリーニング手法として有効に働く可能性が示唆された
</div>

# 実験2の設計

## 目的・位置づけ

実験1（課題・解決手段ともに近い）に対し，**対象物や工程が大きく異なる場合**でも課題文脈を手がかりに候補を整理できるかを確認する補助分析

<div class="warn-box">
⚠ 本実験は花王の半導体分野への進出予測や，技術導入可能性を示すものではない
</div>

## データ定義

| 記号 | 内容 | 役割 |
|---|---|---|
| **S1** | 花王の半導体洗浄関連既知事例 | 参照分野側（対象分野）の課題クラスタ作成に使用 |
| **S0** | 花王のその他洗浄・除去系技術 | S1との比較・フィルタリング用 |
| **C1** | 非半導体系の洗浄・除去技術（C1候補母集団） | 候補抽出の対象集合 |

## 処理の流れ

```
S1 → 7用途クラスタ分類 → 各クラスタのproblem_contextベクトル化
C1（非半導体系） → problem_contextベクトル化
→ コサイン類似度でクラスタ別ランキング（各クラスタ上位20件）
→ ファミリー単位で重複統合（46件） → LLM支援付き著者確認評価
```

# 実験2：S1用途クラスタ

## 7つの用途クラスタ（S1の半導体洗浄既知事例を分類）

| クラスタ | 内容 | クラスタ初出日の役割 |
|---|---|---|
| 接着剤洗浄 | 接着剤残渣の除去 | C1候補抽出の時系列条件 |
| 回路基板洗浄 | プリント基板等の洗浄 | ← |
| CMP後洗浄 | 化学的機械研磨後の残渣除去 | ← |
| フラックス・電子部品洗浄 | フラックス残渣や電子部品洗浄 | ← |
| フォトレジスト・樹脂除去 | レジスト・樹脂の除去 | ← |
| **半導体基板洗浄** | シリコン基板・半導体基板の洗浄 | ← |
| シリコンウェーハ洗浄 | ウェーハ表面の洗浄 | ← |

各クラスタ内S1の最早日を**クラスタ初出日**として設定し，それ以前に存在するC1候補を抽出（時系列制約）

## C1候補母集団の作成条件

1. S1と同一ファミリーは**除外**（既知事例の再発見を避ける）
2. **半導体直接語**を含むものは除外（非半導体系候補に絞る）
3. `substrate` 単独は除外せず**確認用フラグ**として保持

# 実験2：全文テキストベースラインとの比較

## 候補集合の重複・差分

| 比較対象 | 件数 |
|---|---|
| 提案手法（課題文脈ベース） | 46件 |
| 軽量全文テキストベースライン | 56件 |
| **重複するファミリー** | **11件** |
| 提案手法のみの候補 | 35件 |
| ベースラインのみの候補 | 45件 |

**重複11件の内訳**：◎ 4件 / ○ 3件 / △ 2件 / × 2件

<div class="highlight">
重複がわずか11件 → 課題文脈ベースと全文テキストベースは<strong>異なる候補集合</strong>を提示することを示す
</div>

## 解釈

- 全文テキストでは**対象物・用途語・請求項中の表現**が類似度に強く影響
- 課題文脈ベースでは**「何の課題に対応しているか」**に焦点
- どちらが優れているかではなく，**候補集合の傾向が異なる**という補助分析

<div class="warn-box">
⚠ ベースラインのみの候補は未評価のため，精度の優劣として結論づけない
</div>

# 6. 考察・貢献・限界

# 考察：課題・解決手段分離の有効性

## 全文類似との違い

<div class="two-col">

<div>

**全文類似アプローチ**
- 課題・対象物・解決手段・効果が**混在**
- 候補が抽出された理由を説明しにくい
- 異分野間で対象物名・分野名の違いが**強く影響**

実験1検証：全文類似上位ベースライン
→ × が63%，妥当性率**16.7%**

</div>

<div>

**課題文脈分離アプローチ**
- `problem_context`：「何の課題か」
- `solution_context`：「どう解くか」
- 候補抽出理由を**分離して説明可能**

実験1検証：提案手法
→ × が13%，妥当性率**52.2%**

</div>
</div>

## 2種類の実験での確認

| 実験 | タイプ | 結果 |
|---|---|---|
| 実験1（主分析） | 近接解決策型 | U-Netを全体7位・具体的手法2位に抽出，後ろ向き評価と対応 |
| 実験2（補助分析） | 異質解決策型 | 一部クラスタで有効，クラスタ依存性あり |

---

# 本研究の貢献

## 4つの貢献

1. **文脈分離フレームワーク**  
   特許文書を `problem_context` と `solution_context` に分離するフレームワークを提示。候補抽出理由を「課題が近い」「解決手段として○○が提示されている」として説明しやすくした

2. **近接解決策型の主分析（実験1）**  
   U-Netを対象として，医療画像解析→製造欠陥検出への後ろ向き評価を実施。全体ランキング7位（上位10件以内）・具体的手法ランキング2位を確認

3. **異質解決策型の補助分析（実験2）**  
   洗浄・除去系技術から半導体洗浄への候補抽出を探索的に確認。クラスタ設計の重要性を示した

4. **スクリーニング手法としての評価設計**  
   LLM支援付き著者確認評価により，候補の解釈可能性を確認する評価体制を構築

---

# 本研究の限界

| 限界 | 内容 |
|---|---|
| 技術導入可能性 | 候補は専門家確認対象であり，実装・商用化・採用を保証しない |
| LLM抽出の誤差 | `problem_context` 等は中間表現であり，常に正確とは限らない |
| 評価の主観性 | LLM支援付き著者確認評価であり，大規模な第三者専門家評価ではない |
| 特許の限界 | 特許に記載された技術が必ずしも製品化・実装済みとは限らない |
| クラスタ設計依存 | 実験2では用途クラスタや語フィルタ設計が候補品質に大きく影響 |
| ドメイン範囲 | 1ペアのドメイン組み合わせ（実験1）・1企業（実験2）のみ |

## 今後の展望

- **第三者専門家評価**の導入による評価の厳密化
- **他ドメイン**（材料・エネルギー・医薬・化学プロセス等）への適用
- 厳密な**埋め込みベースライン**との比較
- 技術導入可能性評価（実験的検証・市場面分析）との接続

# 7. まとめ

# まとめ

## 本研究の全体像

特許文書を **`problem_context`（課題文脈）** と **`solution_context`（解決手段文脈）** に分離し，課題が近い異分野間で対象分野では低出現な技術手法を専門家確認候補として抽出するスクリーニング手法を提案・検証した

## 実験結果のまとめ

<div class="two-col">

<div>

**実験1（主分析：近接解決策型）**
- A0（医療画像解析）→ B0（製造欠陥検出）
- 過去期間（2015-2018）のみで U-Net を全体**7位**・具体的手法**2位**に抽出
- 将来期間（2019-2024）で B0 に314件出現を確認
- 全文類似ベースラインより候補の**説明可能性が高い**

</div>

<div>

**実験2（補助分析：異質解決策型）**
- 洗浄・除去系技術 → 半導体洗浄
- 46件中 ◎+○ = **16件（35%）** が有効
- 半導体基板洗浄クラスタ：6件全て有効
- **クラスタ設計**が候補品質に大きく影響

</div>
</div>

<div class="result-box">
✅ 課題文脈と解決手段文脈を分離することで，「何の課題が近いのか」「どの解決手段が候補なのか」を説明しやすい専門家確認候補を抽出できることを確認した
</div>

# キーワード

**特許文書** / **課題・解決手段分離** / **クロスドメイン** / **技術候補抽出** / **LLM**

## 参考：GapScoreの再掲

$$\text{GapScore}_m = \text{sim}^{\text{prob}}(A_0, B_0) \times \frac{\text{gap}_m}{\max \text{gap}} \times \log(1 + N_{A,m}) \times \min\!\left(\frac{1}{\text{rate}_{B,m} + \varepsilon}, C\right)$$

## 提案手法の位置づけ

```
膨大な特許文書
      ↓ 課題・解決手段の分離
課題が近い異分野の候補を絞り込み（スクリーニング）
      ↓
専門家確認 → 技術的・事業的評価
```

> 本研究はスクリーニング前段階の手法であり，その後の専門家評価と組み合わせて活用することを前提とする
