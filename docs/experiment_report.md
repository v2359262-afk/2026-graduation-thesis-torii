# 実験レポート：U-Netのクロスドメイン技術候補抽出

## 概要

本レポートでは，Orbis IPから取得した特許データ（A0: 医療画像解析，B0: 製造欠陥検出）を用いて，U-Netの手法ギャップ分析を実施した結果を報告する。

---

## 1. 実験データ

| ラベル | 内容 | 取得元 |
|--------|------|--------|
| A0_Core_combined.xlsx | 医療画像解析分野の特許データ | Orbis IP |
| B0_Core_combined.xlsx | 製造欠陥検出分野の特許データ | Orbis IP |

### データ抽出仕様

- **対象期間**：Filing date 2015年〜2024年
- **U-Net抽出キーワード**：Title または Abstract に "U-Net", "UNet", "nnU-Net", "U-Net++", "UNet++" を含むもの
- **識別基準**：Publication number（B列）が存在するレコードを1件とカウント
- **Family ID**：E列（Family Identifier Simple）に基づく重複除去

---

## 2. 実験プログラム構成

```
extract_patent_data.py   # メインパーサー
find_unet_indices.py     # sharedStrings.xml ストリーム解析（U-Net インデックス検出）
parse_sheet2.py          # sheet2.xml ストリーム解析（特許レコード抽出）
run_extraction.sh        # 実行スクリプト（展開→解析→CSV出力）
analyze_results.py       # 統計計算・グラフ生成
make_results_excel.py    # Excel結果ファイル生成
```

### 技術的特記事項

xlsx ファイルの Central Directory が欠損しているため，Local File Header を直接解析して各コンポーネント（sharedStrings.xml, sheet2.xml）を取得した。sharedStrings.xml（展開後 〜900 MB）は1行形式の XML であったため，ストリーム解析（32 MB チャンク）により U-Net を含む文字列インデックスを特定した。

## 5. 考察

### RQ2（過去期間における低出現候補の抽出）

2015–2018年の過去期間において：
- 医療画像解析（A0）のU-Net出現率：**0.67%**
- 製造欠陥検出（B0）のU-Net出現率：**≈ 0%（1件のみ）**

手法ギャップ $Gap = r_{A0} - r_{B0} \approx 0.67\%$ が確認された。これは，2018年以前のデータから U-Net を「A0分野で先行出現・B0分野で低出現」という候補として抽出できることを示す。

### RQ3（将来期間での出現確認）

2019–2024年の将来評価期間において：
- B0のU-Net出現率：**0.90%**（過去期間の≈0%から上昇）
- 絶対件数：344件（過去期間≈1件から増加）

この結果は，過去期間にU-Netを候補として抽出した場合，後年の対象分野（B0）での出現・増加と対応することを示す後ろ向き評価の根拠となる。

### 課題・解決手段分離の意義（RQ1）

医療画像解析と製造欠陥検出は，「画像中の異常・欠陥領域を検出・分割する」という課題構造が近い。この課題類似性を前提として，医療画像解析で高出現（4.00%）・製造欠陥検出で低出現（0.81%）という構造が確認された。このギャップを「手法ギャップ」として定義することで，候補抽出の理由を「課題が近い異分野間での解決手段の出現率差」として説明可能にした。

## 6. 限界

1. **データ重複**：A0_Core_combined.xlsx はOrbis IPの複数検索結果を結合したファイルであり，同一発明の複数国出願が重複計上されている。絶対数値は論文記載値と異なるが，出現率の傾向は整合している。
2. **抽出精度**：U-Netの表記揺れ（"U-Net", "UNet", "nnU-Net", "U-Net++", "UNet++"）をキーワード一致で検出しており，関連構造（例：マルチスケールエンコーダ・デコーダ）は補足されない。
3. **特許データの限界**：U-Netは論文・オープンソースで広く利用されているが，特許として出願されていない事例も多い。特許出現率の低さは必ずしも技術未利用を意味しない。
